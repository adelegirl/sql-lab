## 📚 内容概览（v0.14）

| 页面 | 主题 | 题量 | 说明 |
|---|---|---|---|
| **index** | SELECT 查询练习 | 50 题 | SQLZoo 中文改编，SELECT 从入门到进阶 |
| **ddl** | DDL 数据定义 | 15 题 | CREATE / ALTER / DROP，覆盖约束、索引、视图 |
| **dml** | DML 数据操纵 | 15 题 | INSERT / UPDATE / DELETE，含事务与批量操作 |
| **dcl** | DCL 数据控制 | 15 题 | GRANT / REVOKE / ROLE，虚拟权限矩阵模拟 |
| **algebra** | 关系代数 | 24 题 | σ / π / 集合运算 / 连接 / 除法，符号语言 ↔ SQL 双轨对照 |
| **nf** | 范式训练 | 17 题 | 求候选码 · 判最高范式 · 分解到目标范式（含无损与依赖保持双验证） |
| **er** | ER 建模（拖拽式概念建模） | 12 题 | 读文字需求 → 拖拽实体/关系/连线绘制 ER 图 → 图结构等价判分（实体 / 属性 / 主键 / 关系 / 基数） |

**共 148 题**，覆盖 SQL 四大子语言 + 关系代数符号语言 + 关系模式规范化理论 + ER 建模到关系模式转换。

**统一业务场景**：
- **SQL 四页**（SELECT / DDL / DML / DCL）共用 `employee` / `department` / `project` 三张表；
- **关系代数页 + 范式页**同样沿用该场景，但表名、属性名全部用**纯字母符号**（R / S / T / A, B, C…）呈现，避免学生"看到熟悉表名就秒选答案"，强制走抽象思维通道。
- **ER 建模页**独立使用 12 个小型独立业务场景（图书 / 员工 / 产品 / 部门 / 出版社 / 客户订单 / 学生选课 / 作者图书 / 订单明细 / 员工家属 / 人员特化 / 车辆特化），每个场景聚焦一种 ER 模式（单实体 / 一对多 / 多对多 / 弱实体 / 类表继承）。

## 🧮 关系代数页特别说明（v0.11）

关系代数页与 SQL 四页判分机制不同：
- **SQL 四页**：靠 sql.js（SQLite WebAssembly）实际执行 SQL 比对结果集
- **关系代数页**：纯 JS **AST 抽象语法树**判分 —— 学生拖拽运算符构建表达式，页面解析 AST 后与答案对比，**不依赖任何 SQL 引擎**

**覆盖运算符**：σ 选择、π 投影、⋈ 连接、÷ 除法、∪ 并、∩ 交、− 差

**交互亮点**：
- **目标横条瘦身**：蓝条只展示自然语言目标，参考答案藏在折叠区，避免贴脸暴露
- **弹窗式参数配置**：σ 条件 / π 字段 / 二元运算操作数（表 or 已定义中间关系 F）全部弹窗式点选
- **中间结果复用**：定义好 F1 之后，后续运算符的第二操作数可直接引用 F1，不必每次重写完整子树

## 🔍 范式训练页特别说明（v0.12）

范式页采用**算法判分**，让学生真正理解规范化理论的每一步：

- **三步作答**：每题依次求候选码 → 选最高范式 → 分解到目标范式（1NF / 2NF 只判前两步，3NF / BCNF 需完整走完三步）
- **五算法内核**：`attrClosure` 属性闭包 · `findCandidateKeys` 候选码 · `checkNF` 范式判定 · `losslessDecomp` 无损分解（表格法） · `depPreserving` 依赖保持
- **属性闭包计算器**：每题上方常驻小工具，可实时计算任意属性集 X 关于 F 的闭包 X⁺，**辅助推演不消耗尝试次数**
- **候选码输入宽松解析**：支持 `(A,B), (A,C)` / `AB, AC` / 分号顿号换行等多种写法，忽略大小写和括号
- **分解判分双通道**：先按严格集合匹配；不匹配再宽松验收（覆盖全属性 + 无损 + 每 Ri 达到目标范式），学生只要给出等价的合法分解都能通过
- **17 题分层设计**：
  - 🌱 入门 5 题：单候选码，聚焦"求码 + 判范式"两步，含 BCNF / 3NF-not-BCNF / 2NF / 1NF 全谱系
  - 📚 中等 8 题：4~5 属性，含多候选码、传递依赖，全部分解到 3NF
  - 🎯 综合 4 题：分解到 BCNF + 无损 & 依赖保持双验证，含教学负面案例（BCNF 强推丢失依赖）

**教学负面案例**：N3_2（R=ABC, F={AB→C, C→B}）分解到 BCNF 会丢失依赖 AB→C，用来说明"为什么教材推荐 3NF 而非硬推 BCNF"。

## 🔷 ER 建模页特别说明（v0.14 拖拽式概念建模器）

**v0.14 大改版**：从 v0.13 的"看 ER 图 → 写 CREATE TABLE"（P2 关系模式转换）转向真正的 **P1 概念建模**——学生读完文字需求后，通过**拖拽实体、关系、连线**在画布上直接绘制 ER 图，回到白板画图的直觉体感，与工作流可视化编辑器（如扣子）的手感一致。

- **建模器实现**：**原生 SVG + Pointer Events**（pointerdown / pointermove / pointerup + setPointerCapture）。**零外部依赖**，不引 GoJS / drawio / jsPlumb / D3 / Fabric，纯前端离线运行；桌面鼠标 + 移动触屏天然兼容
- **组件面板**（左侧）：
  - ➕ **实体（矩形）**：普通强实体，蓝色边框
  - ➕ **弱实体（虚框）**：紫色双层虚线矩形，用于依附强实体的弱实体
  - ➕ **关系（菱形）**：橙色菱形，可加自己的属性
  - ➕ **弱关系（虚菱）**：黄色虚线菱形，用于识别关系
  - 🔗 **连线模式**：进入后依次点两个节点即完成连线
- **画布交互**：
  - 每个节点可自由拖动，位置持久化到 localStorage（每题独立缓存）
  - 双击节点改名，选中节点后浮出"改名 / 编辑属性 / 转弱实体 / 删除"按钮
  - 属性编辑器行式布局：属性名 + PK 勾选 + 删除
  - 连线中点显示圆形基数标签，点击循环切换 `1 → N → M`
- **判分机制**：**图结构等价对比**，四维度加权：
  1. **实体完备性**：按名称集合对比，内置**同义词表**（学生/Student/stu、员工/employee/职工、图书/book、部门/department 等；每题涵盖 40+ 常见同义词）
  2. **实体属性完备性**：忽略顺序，主键必须标注；属性名支持中英文和常见变体同义
  3. **关系存在性**：按 `{entity1, entity2, cardinality}` 三元组集合对比（entity1/2 无序，可以反向画）
  4. **基数正确性**：`1:1 / 1:N / M:N` 三档，`N:1` 自动归一为 `1:N`
- **反馈粒度**：三档结果 —— **完全正确 / 部分正确 / 结构错误**，附带每一条**可操作**的反馈（"缺少实体：课程"、"实体「图书」缺少属性：定价"、"关系「选修」基数应为 M:N，实际为 1:N"）
- **12 题分层设计**：
  - 🌱 入门 3 题：单实体（图书 / 员工 / 产品），练"实体-属性-主键"三件套的操作手感
  - 🔗 一对多 3 题：部门-员工 / 出版社-图书 / 客户-订单，练两实体 + 菱形 + 1:N 结构
  - 🕸️ 多对多 2 题：学生-课程（含成绩） / 作者-图书（含贡献排名），练 M:N + 关系带属性
  - 🪶 弱实体 2 题：订单-订单明细 / 员工-家属，练弱实体（虚框） + 识别关系（虚菱）
  - 🧬 继承 2 题：人员→学生/教师 / 车辆→轿车/卡车，练 3 实体 + 2 个"特化 / ISA"菱形关系
- **教师视角**：学生真的在画图（而非"填 SQL 空"），可以从画布上看到学生对概念模型的理解层级；判分器专注于图结构本身，不苛求命名字面
- **自测**：页面加载时自动跑 **42 组** 判分函数自测（`[ER拖拽判分自测] PASS 42, FAIL 0 ✅`）覆盖同义识别、基数归一、单实体/一对多/多对多/弱实体/继承五类的参考图自通过 + 5 类学生变体测试

**分层提示**：每题 3 条上限，从"读图思路" → "结构骨架" → "画布操作"逐层递进，永远不甩答案。

**已知限制**：
- 三元及以上高维关系判分按两两拆分处理，理论上足以覆盖本课程的经典模式，但复杂场景（如 N-ary 关系必须整体判分）暂未支持
- 弱实体标注（虚框）是判分维度，学生需要用"➕ 弱实体"按钮而不是普通"➕ 实体"；关系带自身属性（如 N1 "成绩"）目前作为"可选加分项"不强制
- 触屏浏览器兼容 Pointer Events，但小屏（≤ 800px）下需要横向滚动画布查看全景

## 🛠️ 技术栈

- **sql.js**（SQLite WebAssembly）—— SQL 四页 + ER 建模页在浏览器内真跑 SQL
- **mermaid.js v10** —— ER 建模页把 `erDiagram` 源码渲染成可读的 ER 图
- **AST 判分器**（关系代数页自研）—— 纯 JS 解析符号表达式树并等价比对
- **五算法内核**（范式页自研）—— 属性闭包 / 候选码 / 范式判定 / 无损分解 / 依赖保持
- **结构判分器**（ER 页自研）—— PRAGMA 抽 schema + 7 维等价对比 + 教学友好宽容口径
- 纯 HTML + CSS + JS，零依赖、零后端
- 进度保存在 `localStorage`（各页独立 key，三态追踪：通过 / 尝试 / 看过答案）

**DCL 特别说明**：SQLite 本身不支持 GRANT / REVOKE / ROLE。DCL 页用**虚拟权限矩阵**模拟执行，页面顶部有诚实 banner 提示"本页 DCL SQL 不真跑在 SQLite"。

**关系代数特别说明**：本页无任何 SQL 执行，纯粹训练学生对 σ / π / ⋈ / ÷ 等符号的组合理解，与 SQL 页互补——SQL 页练"怎么把自然语言翻译成 SQL"，关系代数页练"怎么把自然语言翻译成符号语言"，**双轨训练**学生才能真正理解查询的关系本质。

**范式特别说明**：本页也不跑 SQL，用五算法内核直接对关系模式 R + 函数依赖集 F 做规范化推演；判分器在学生输入分解方案时会做无损分解表格法 + 依赖保持等价性双重校验，允许任何等价的合法分解写法通过。

**ER 建模特别说明**：本页把设计端到实现端打通——学生看到的是 mermaid ER 图，写出的是 CREATE TABLE，判分对比的是最终 schema 结构。这条链路对应现实工作流：需求分析 → 概念模型 → 逻辑模型 → 物理模型；学生不再是"背知识点"，而是"做设计"。

## 🎓 教学设计原则

- **4 口令 + 3 抽象**：
  - `查询 / 修改结构 / 修改数据 / 权限` —— 对应 SELECT / DDL / DML / DCL
  - `抽象查询` —— 关系代数符号（σ / π / ⋈ / ÷ / ∪ / ∩ / −）
  - `抽象设计` —— 关系规范化（候选码 / 1~BCNF / 无损分解 / 依赖保持）
  - `具象设计` —— ER 建模到关系模式转换（单实体 / 一对多 / 多对多 / 弱实体 / 类表继承）
- **2 铁律**：
  1. 每题必给验收标准
  2. 提示（hints）上限 3 条，不锁难度
- **四轨互补**：SQL 四页练"写"，关系代数页练"理解查询"，范式页练"理解设计（规范化）"，ER 页练"落地设计（建模到表结构）"；学生四条路都走过，才算真正吃透关系数据库
- **主战场**：机房 + 电脑，浏览器直接打开链接即可
