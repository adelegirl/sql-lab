## 🧮 关系代数页特别说明（v0.11 新增）

关系代数页与 SQL 四页判分机制不同：
- **SQL 四页**：靠 sql.js（SQLite WebAssembly）实际执行 SQL 比对结果集
- **关系代数页**：纯 JS **AST 抽象语法树**判分 —— 学生拖拽运算符构建表达式，页面解析 AST 后与答案对比，**不依赖任何 SQL 引擎**

**覆盖运算符**：σ 选择、π 投影、⋈ 连接、÷ 除法、∪ 并、∩ 交、− 差

**交互亮点**：
- **目标横条瘦身**：蓝条只展示自然语言目标，参考答案藏在折叠区，避免贴脸暴露
- **弹窗式参数配置**：σ 条件 / π 字段 / 二元运算操作数（表 or 已定义中间关系 F）全部弹窗式点选
- **中间结果复用**：定义好 F1 之后，后续运算符的第二操作数可直接引用 F1，不必每次重写完整子树

## 🛠️ 技术栈

- **sql.js**（SQLite WebAssembly）—— SQL 四页在浏览器内真跑 SQL
- **AST 判分器**（关系代数页自研）—— 纯 JS 解析符号表达式树并等价比对
- 纯 HTML + CSS + JS，零依赖、零后端
- 进度保存在 `localStorage`（各页独立 key，三态追踪：通过 / 尝试 / 看过答案）

**DCL 特别说明**：SQLite 本身不支持 GRANT / REVOKE / ROLE。DCL 页用**虚拟权限矩阵**模拟执行，页面顶部有诚实 banner 提示"本页 DCL SQL 不真跑在 SQLite"。

**关系代数特别说明**：本页无任何 SQL 执行，纯粹训练学生对 σ / π / ⋈ / ÷ 等符号的组合理解，与 SQL 页互补——SQL 页练"怎么把自然语言翻译成 SQL"，关系代数页练"怎么把自然语言翻译成符号语言"，**双轨训练**学生才能真正理解查询的关系本质。

## 🎓 教学设计原则

- **4 口令 + 1 抽象**：
  - `查询 / 修改结构 / 修改数据 / 权限` —— 对应 SELECT / DDL / DML / DCL
  - `抽象` —— 关系代数符号（σ / π / ⋈ / ÷ / ∪ / ∩ / −）
- **2 铁律**：
  1. 每题必给验收标准
  2. 提示（hints）上限 3 条，不锁难度
- **双轨互补**：SQL 四页练"写"，关系代数页练"理解"；SQL 页看具体表名，关系代数页看抽象字母；学生两条路都走过，才算真正吃透关系查询
- **主战场**：机房 + 电脑，浏览器直接打开链接即可
